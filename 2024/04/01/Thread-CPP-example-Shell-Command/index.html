多线程的简单概念

什么是线程？

进程：是一个正在执行的程序，是资源分配的最小单位。
确切的来说，当一个程序进入内存运行，即变成一个进程，进程是处于运行过程中的程序，并且具有一定独立功能。
每一个进程执行都有一个执行顺序，该顺序是一个执行路径，或者叫一个控制单元。进程中的程序是按照一定顺序去执行的。
如何让一个进程中的事情同时发生呢？多线程的概念
线程：轻量级的进程，程序执行的基本单位，进程是为了分配资源，执行的是线程。
是进程中的一个执行单元，负责当前进程中程序的执行，一个进程中至少有一个线程。一个进程中是可以有多个线程的，这个应用程序也可以称之为多线程程序。
线程之间的关系：像小孩抢糖吃，所有的线程共享进程的资源，也能够进行通信。

简而言之：一个程序运行后至少有一个进程，一个进程中包含多个线程。

线程是操作系统能够进行运算调度的最小单位，它被包含在进程之中，是进程中实际运作单位。一条线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务。

什么是线程安全？
当多个线程访问某个方法时，不管你通过怎样的调用方式、或者说这些线程如何交替地执行，我们在主程序中不需要去做任何的同步，这个类的结果行为都是我们设想的正确行为，那么我们就可以说这个类是线程安全的。
当多个线程访问那个方法的时候，可能会面对线程间的互斥，临界区的访问这些问题，所以无法保证最后线程运行的结果和我们预期的一样。

并行和并发

单核和多核
并发（Concurrency）：同一时刻只有一条执行，但是多个进程之间快速的切换。
通过一种算法将CPU资源合理地分配给多个任务，当一个任务执行I/O操作时，CPU 可以转而执行其他的任务，等到I/O操作完成以后，或者新的任务遇到I/O操作时，CPU再回到原来的任务继续执行。
一般是单核CPU的执行指令的方式。
并行（Parallelism）
针对多核CPU所提出的。
每个CPU都能够独立地执行一个任务，而且多个核心之间不会相互干扰。在不同核心上执行的多个任务，是真正的同时运行。

并发针对单核 CPU 而言，它指的是 CPU 交替执行不同任务的能力；并行针对多核 CPU 而言，它指的是多个核心同时执行多个任务的能力。

单核 CPU 只能并发，无法并行；换句话说，并行只可能发生在多核 CPU 中。
在多核 CPU 中，并发和并行一般都会同时存在，它们都是提高 CPU 处理任务能力的重要手段。

同步和异步

同步：彼此之间有依赖的调用不应该“同时发生”，而同步就是要阻止那些“同时发生”的事情。
异步：任何两个彼此独立的操作都是异步的，表情事件独立的发生。
同步和异步能够互相转换。

多线程的优势

多处理器中开发程序的并行性。
再等待慢速IO操作时，程序可以执行其他操作，提高了并发性。
模块化的编程，能更清晰地表达程序之间的关系
占用较少的系统资源。

线程的生命周期

线程中首先运行的main函数，就是初始线程或者主线程。主函数返回的时候就是进程结束。如果只是想退出主线程，使用线程退出函数 pthread_exit 函数。
主线程接受的参数是 argc 和 argv，普通的线程只有一个 void* 。
绝大多数情况下，主线程在默认堆栈上运行，这个堆栈可以增长到足够的长度，而普通线程的堆栈是受到限制的，一旦溢出就会产生错误。
主线程随着进程的创建而创建，是由内核来做的。新线程的运行可能在 pthread_create 返回之前就执行完毕了。

线程的状态和回收

状态

状态 含义
就绪 线程能够运行，但是在等待可用的处理器 线程刚被创建的时候，或者解除阻塞的时候。
运行 线程正在运行，在多核系统中，可能同时有多个线程在运行
阻塞 线程在等待处理器以外的其他条件 互斥量，无法加锁，缺乏资源等，和处理器无关的缺乏条件。
终止 线程从启动函数中返回，或者调用 pthread_exit 函数，或者被取消

回收

分离一个正在运行的线程并不影响它。仅仅是通知系统当线程结束的时候，所占据的资源可以被回收。一个没有被分离的线程在终止时会保留它的虚拟内存，包括他们的堆栈和其他系统资源。有时这种线程被称为“僵尸线程”。
创建线程时默认时非分离的。
如果线程具有分离属性，那么线程终止的时候会被立刻回收，回收将释放掉所有在线程终止时未释放的系统资源和进程资源，包括保存线程返回值的内存空间、堆栈、保存寄存器的内存空间等。
终止被分离的线程会释放所有的系统资源，但是你必须释放由该线程占有的系统资源。由malloc 或者 mmap 分配的内存可以在任何时候由任何线程释放。
条件变量、互斥量、信号灯可以由任何线程销毁，只要他们被解锁了或者没有线程等待，但是只有互斥量的主人才能解锁它，所以在线程终止前，你需要解锁互斥量。
